# 시연 체크리스트

NeoSquare 시연은 `계정 2개`, `브라우저 2개(또는 일반 창 + 시크릿 창)` 기준으로 준비하는 것이 가장 안정적입니다.

## 1. 시연 전 준비

- backend 실행 확인
- frontend 실행 확인
- `/api/health` 응답 확인
- 더미 계정 또는 직접 만든 계정 2개 준비
- 카메라/마이크 권한 허용
- 필요하면 두 브라우저 창을 나란히 배치

권장 계정 조합:
- A: `mina@neosquare.local`
- B: `jisu@neosquare.local`

## 2. 기본 시연 순서

### 1. 로그인과 로비 진입
- A 로그인
- B 로그인
- 로비 페이지 진입
- Phaser 로비 이동과 공개 채팅 확인

### 2. 공간 이동과 상호작용
- `Study Lounge` 또는 `Mentoring Room` 공간 입장
- 다른 아바타 클릭
- 채팅 / 멘토링 요청 / 예약 제안 버튼 확인

### 3. 멘토링 요청 흐름
- A가 B에게 멘토링 요청 전송
- B가 받은 요청에서 수락
- A 또는 B가 알림/목록에서 세션 입장

### 4. WebRTC 준비 흐름
- 카메라/마이크 준비 버튼 클릭
- 권한 허용
- 연결 상태, 시그널링 상태, 최근 signal 표시 확인
- 필요하면 연결 끊김 후 `연결 다시 시도` 버튼 시연

### 5. 세션 종료와 요청 기반 피드백
- 세션 종료
- 로비 복귀 후 피드백 카드 확인
- 평점 / 요약 / 피드백 저장
- 히스토리 패널에서 저장 결과 재조회

### 6. 예약 흐름
- A가 B에게 예약 생성
- B가 받은 예약에서 수락
- 예약 시간이 입장 가능 범위에 들어오면 세션 입장
- 종료 후 예약 피드백 저장

### 7. 알림 흐름
- 요청 수락 알림 확인
- 예약 수락 알림 확인
- 읽음 처리 / 전체 읽음 처리 확인

### 8. 스터디 세션 흐름
- `Study Lounge` 입장
- 스터디 세션 생성
- 다른 계정으로 참가
- 스터디 세션 페이지 입장
- 스터디 전용 채팅 확인

## 3. 시연 중 자주 보는 확인 포인트

- 로비/공간 이동이 실제 상호작용으로 이어지는가
- 요청/예약/알림이 새로고침 후에도 서버 기준으로 유지되는가
- 세션 종료 후 피드백이 다시 조회되는가
- 같은 계정이 아니라 다른 계정 상호작용으로 흐름이 닫히는가

## 4. 실패 시 빠른 점검

### 로그인/회원가입이 안 될 때
- backend가 떠 있는지 확인
- 브라우저 localStorage 토큰이 꼬였으면 로그아웃 후 재로그인

### 로비 채팅/위치가 안 보일 때
- 두 사용자가 같은 공간에 있는지 확인
- `/ws` 연결이 끊기지 않았는지 브라우저 콘솔 확인

### 영상 연결이 안 될 때
- 카메라/마이크 권한 허용 확인
- `VITE_WEBRTC_ICE_SERVERS` 설정 확인
- 같은 네트워크/브라우저 정책 이슈면 `연결 다시 시도`

### 예약 입장 버튼이 안 보일 때
- 예약 상태가 `ACCEPTED`인지 확인
- 예약 시간이 입장 가능 범위인지 확인

## 5. 발표 때 강조하면 좋은 포인트

- 멘토링 전용 서비스가 아니라 공간 기반 커뮤니티라는 점
- 요청/예약/스터디가 각각 별도 흐름이지만 하나의 사용자 경험으로 연결된다는 점
- fallback에서 서버 저장형 구조로 전환했다는 점
- WebSocket senderId 신뢰 제거, WebRTC 재시도, 알림/피드백까지 이어지는 lifecycle을 직접 정리했다는 점
