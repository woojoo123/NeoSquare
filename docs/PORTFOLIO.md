# NeoSquare Portfolio Summary

## 한 줄 소개
Phaser 기반 가상 로비와 Spring Boot 백엔드를 결합해, 멘토링을 포함한 커뮤니티 상호작용을 서버 저장형 흐름으로 연결한 메타버스형 커뮤니티 프로젝트입니다.

## 2~3문장 소개
NeoSquare는 사람들이 가상 공간 안에서 만나고 대화하며, 필요할 때 멘토링 요청과 예약, 세션 진입, 피드백까지 이어갈 수 있도록 설계한 프로젝트입니다. 단순한 멘토링 매칭 서비스가 아니라, 공간 인터페이스를 통해 연결 경험을 더 직관적으로 만드는 데 초점을 두었습니다. 프론트의 임시 fallback 구조를 점진적으로 제거하고, 인증·요청·예약·피드백·알림 흐름을 실제 서버 저장 기반으로 정리했습니다.

## 핵심 기술 포인트 3개
- Spring Boot 기반 REST API와 JWT 인증, WebSocket handshake 사용자 식별을 연결해 REST와 realtime 흐름의 사용자 기준을 일관되게 맞췄습니다.
- 멘토링 요청, 예약, 세션 종료, 피드백, 알림을 각각 별도 도메인으로 분리하면서도 하나의 사용자 흐름으로 이어지도록 구현했습니다.
- React + Phaser + WebRTC 구조를 조합해 로비 경험, 세션 진입, signaling, 영상통화 준비 흐름까지 한 프로젝트 안에서 연결했습니다.

## 차별점 3개
- 멘토링 자체보다 "가상 공간 안에서 연결이 일어나는 경험"을 먼저 두고, 멘토링을 그 안의 활동으로 배치한 구조입니다.
- 요청 기반 세션과 예약 기반 세션을 모두 지원해 즉시성 있는 연결과 시간 예약형 연결을 함께 보여줍니다.
- 프론트 데모용 fallback에서 서버 저장형 구조로 옮겨가며, 실제 서비스화에 필요한 데이터 흐름 정리를 직접 수행한 프로젝트입니다.

## 발표/소개 시 강조 포인트
- 단순 CRUD가 아니라 인증, 실시간 연결, 세션 lifecycle, 알림, 피드백까지 이어지는 사용자 흐름을 설계한 점
- WebSocket senderId를 서버 기준 사용자 식별로 보강해 운영형 보안 관점까지 신경 쓴 점
- Phaser 로비와 WebRTC 세션 페이지를 함께 사용해 "커뮤니티 공간 + 활동 흐름"을 묶은 점
